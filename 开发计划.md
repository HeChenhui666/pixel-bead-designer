# 拼豆图纸生成器 - 深度开发计划与技术实现文档

## 1. 项目定位与目标

目标是构建一款支持 Android/iOS 原生 App 与微信小程序的拼豆图纸生成器，核心体验包括：

* 用户上传照片后自动生成拼豆网格图纸
* 自动匹配常见品牌拼豆色号
* 支持缩放、拖拽、编辑、色块替换
* 支持图纸导出、长图分享与历史记录

核心技术栈为：

* **UniApp x**：用于 App 端原生编译与高性能 Canvas 处理
* **Vue 3 / UniApp Vue 语法**：用于业务 UI、组件与页面结构
* **UTs**：用于性能敏感的图像处理与色彩计算逻辑
* **Pinia**：用于状态管理与跨页共享
* **uni-app API**：用于文件、存储、Canvas、Worker、分享等平台能力

> 设计原则：逻辑层尽量通用、UI 层条件编译、多端共用数据模型、性能操作优先 UTs 原生执行。


## 2. 总体架构与模块划分

应用分为四个核心模块：

1. **图像输入与预处理模块**
2. **颜色映射与拼豆规则引擎**
3. **图纸渲染与交互模块**
4. **导出、历史与平台适配模块**

模块边界清晰，便于 UniApp x 与标准 Vue 语法混合编写。

## 3. 详细技术实现

### 3.1 图像输入与预处理

#### 3.1.1 图片选择与兼容方案

* APP：优先使用 `uni.chooseImage({ count: 1, sizeType: ['original', 'compressed'] })`
* 小程序：兼容 `wx.chooseImage` 或 `uni.chooseImage`。
* H5：使用 `<input type="file" />` 做扫描预览。

#### 3.1.2 压缩与高宽限制

目标：避免直接处理 4K 大图，确保后续像素分析可控。

* 先读取图片实际尺寸
* 目标最大边长控制在 `1024px` 或 `1200px`
* 保持宽高比，生成 `workWidth/ workHeight`
* 生成低分 `previewImage` 用于页面展示

#### 3.1.3 Canvas 预处理

* `APP-PLUS`：优先使用 UTs 原生 Bitmap 或离屏 Canvas 接口
* `MP-WEIXIN`：使用小程序 `wx.createCanvasContext` + `canvasToTempFilePath`
* `H5`：使用标准 Canvas2D `drawImage`

#### 3.1.4 像素化与区域采样

算法流程：

1. 计算目标网格尺寸 `gridWidth`, `gridHeight`
2. 计算区域块大小 `blockW = workWidth / gridWidth`, `blockH = workHeight / gridHeight`
3. 遍历每个单元格，读取该区域的像素
4. 统计像素颜色
   * 平均色
   * 中位色（可选）
   * 颜色直方图/主色提取
5. 返回 `gridColorData`

#### 3.1.5 性能优化

* 批量读取像素到 `Uint8ClampedArray`
* 在 UTs 中避免频繁创建数组对象
* 使用 `for` 循环替代高阶迭代函数
* 必要时将计算分片到 Worker 线程
* 处理完毕后及时 `null` 掉临时对象

#### 3.1.6 示例伪代码（UTs）

```typescript
import { createImageDataFromCanvas } from './canvas-util.uts';

export function pixelateImage(imagePath: string, gridWidth: number, gridHeight: number) {
  const { width, height, pixels } = createImageDataFromCanvas(imagePath);
  const blockW = width / gridWidth;
  const blockH = height / gridHeight;
  const cells = [];

  for (let y = 0; y < gridHeight; y++) {
    const row = [];
    for (let x = 0; x < gridWidth; x++) {
      const avg = sampleBlockAverage(pixels, width, height, x, y, blockW, blockH);
      row.push(avg);
    }
    cells.push(row);
  }

  return { width, height, cells };
}
```

### 3.2 颜色映射引擎

#### 3.2.1 色卡管理

* 内置色卡：Perler、Hama、Fuse、官方通用色号
* 支持用户自定义色卡
* 每个色卡条目包含：`id`、`name`、`brand`、`r/g/b`、`hex`、`translate`、`stock`

#### 3.2.2 色差计算

推荐实现两层选取策略：

1. 先使用快速 RGB 欧氏距离筛选 TopN 色号
2. 再使用 `Lab -> CIEDE2000` 精选最优颜色

简易色差公式：

```typescript
function rgbDistance(a, b) {
  return (a.r-b.r)**2 + (a.g-b.g)**2 + (a.b-b.b)**2;
}
```

进阶公式（推荐）：

* `rgb2lab(r,g,b)`
* `ciede2000(lab1, lab2)`

#### 3.2.3 KD-Tree 与加速

* 对 `colors.json` 做离线预处理，构建 KD-Tree 或八叉树
* 对于每个像素块，先在空间树中查询最近邻 16 个颜色
* 再对 16 个候选色进行 CIEDE2000 精选

该策略兼顾速度与视觉稳定性。

#### 3.2.4 色卡匹配流程

1. 对单元格颜色进行 `n` 次采样
2. 计算平均 `rgb`
3. 先用 `rgbDistance` 获取候选集
4. 对候选集做 `Lab` 转换和 `CIEDE2000` 评分
5. 选择最小色差
6. 记录 `colorId`、`colorHex`、`count`

#### 3.2.5 参考开源实现

* `Zippland/perler-beads`：颜色查找与拼豆矩阵数据结构
* `liangdabiao/perlerBeadsApplet`：小程序导出、图纸显示与交互体验
* `DanZai233/PixelBead`：色块统计与配色表生成

### 3.3 图纸渲染与交互

#### 3.3.1 Canvas 渲染策略

渲染分三层：

1. `color layer`：单元格填充颜色
2. `grid layer`：网格线
3. `overlay layer`：索引、符号、文字标签

建议使用 `Offscreen Canvas` 或离屏缓存：

* 先绘制单元格到 `color layer`
* 将 `grid line` 与 `symbol` 分开放置
* 缩放/拖拽只对容器做 `transform`
* 导出时再完整重绘一次高分辨率图

#### 3.3.2 交互功能

* 缩放：双指捏合、鼠标滚轮
* 平移：拖拽、右键移动
* 单击选中：切换单元格颜色
* 斜边/中心线切换：辅助线显示
* 撤销/重做：基于状态快照

#### 3.3.3 视图层与条件编译

* `APP-PLUS`：原生性能优先 Canvas 直绘
* `MP-WEIXIN`：小程序 Canvas + `cover-view` 兼容层
* `H5`：标准 Canvas + `pointer events`

#### 3.3.4 视图优化技巧

* 仅在内容变化时更新 Canvas
* 用 `requestAnimationFrame` 执行重绘
* 文字/符号不必要时禁用，减少绘制成本
* 通过 `scale` 变换显示预览，导出时再绘制真实尺寸

#### 3.3.5 样式与组件

* `GridCanvas.vue`：负责绘制与交互事件
* `ColorPalette.vue`：色卡选择与过滤
* `ToolBar.vue`：网格参数、导出入口、回退按钮
* `PreviewPanel.vue`：渲染结果缩略图与统计数据

### 3.4 导出与存储

#### 3.4.1 导出方案

* 图片导出：`uni.canvasToTempFilePath`、`canvasToTempFilePath`、`toDataURL`
* 长图导出：先绘制“图纸 + 配色表”到一张超长 Canvas，再生成图片
* PDF 导出：可选 `jsPDF`，但 App 端更推荐导出 PNG 后交给系统分享

#### 3.4.2 历史记录与本地存储

* 使用 `uni.setStorageSync('history_list', history)` 保存小规模历史
* 使用 `uni.getFileSystemManager()` 保存本地缩略图文件
* 历史数据项包含：
  * `title`, `date`, `grid`, `palette`, `stats`, `thumbPath`, `sourcePath`

#### 3.4.3 分享与平台差异

* App 端：`uni.share` / `plus.share` 做系统分享
* 小程序：生成临时文件后 `wx.shareAppMessage`
* H5：生成图片后直接下载或复制到剪贴板

## 4. 多端适配与条件编译细节

### 4.1 UniApp x 常见条件编译场景

* `#ifdef APP-PLUS`
* `#ifdef MP-WEIXIN`
* `#ifdef H5`
* `#ifdef APP-PLUS-NVUE`

### 4.2 App 端原生 UTs 优化点

* 优先使用 `UTSBitmap` 或离屏 `UTSCanvas`
* 直接读取像素数组，避免转换回 Base64
* 复杂计算尽量在 UTs 中完成，避免 JS 和原生之间频繁调用

### 4.3 小程序端兼容策略

* 小程序 Canvas 不支持所有标准 Canvas API，使用 `wx.createCanvasContext`
* 复杂计算使用 `Worker`，避免 UI 线程卡顿
* 小程序 `cover-view` 层与 `canvas` 层叠加时，注意遮罩与交互显示

### 4.4 H5 端测试方案

* 用 H5 端快速验证像素化算法与渲染效果
* 兼容 `canvas.toDataURL()`、`Image` 加载与文件选择逻辑
* 仅作为算法验证与 UI 微调，不作为最终发布目标

## 7. 风险与应对

### 7.1 UTs 与第三方库兼容风险

* 风险：某些 NPM 库无法在 UTs 中正常运行
* 应对：将核心算法用纯函数或 UTs 实现，避免依赖 DOM/浏览器 API；必要时写 Polyfill 或降级支持。

### 7.2 小程序 Canvas 性能风险

* 风险：复杂绘制导致 MP-WEIXIN 卡顿，无法同时绘制大尺寸图纸
* 应对：使用 Worker 计算、分块渲染、只在导出时执行高分辨率重绘。

### 7.3 颜色偏差与用户预期差距

* 风险：色差导致用户对拼豆结果不满意。
* 应对：提供“真实色号预览”与“自定义校准”，允许用户微调色卡 RGB 值并保存自定义方案。

### 7.4 数据存储与历史膨胀

* 风险：历史记录占用过多空间
* 应对：只保存关键元数据与缩略图；定期清理旧记录；对导出图使用临时缓存。

## 8. 关键技术实现细节

### 8.1 UTs 性能关键点

* 优先在 UTs 中处理像素矩阵，而非 JS 回调
* 避免 `Array.map` / `Array.filter` 在像素循环中产生临时对象
* 使用 `Uint8ClampedArray` 和 `ArrayBuffer` 做批量像素读取
* 直接将 Canvas 输出写入本地文件，避免中间 Base64 转换

### 8.2 颜色匹配优化

* 先用「粗选 + 精选」策略降低色卡匹配成本
* 若色卡数量 < 200，可直接做一次线性扫描
* 若色卡 > 300，建议做 KD-Tree/八叉树索引

### 8.3 导出高质量长图

* 先在离屏 Canvas 内绘制带配色表的长图
* 导出时按目标 DPI 缩放，并考虑生成 `@2x` 图
* 对文字、色块使用 `stroke` / `fill` 分层绘制，保证导出清晰

### 8.4 兼容性控制

* 统一 `utils/platform.ts` 返回当前平台标签
* 在 Vue 页面中使用 `#ifdef` 条件编译分离平台代码
* 对于不支持的 API，提供降级方案或提示用户
